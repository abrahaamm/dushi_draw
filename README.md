# 笃实抽奖系统

这是一个用于现场大屏抽奖的静态网页项目。系统分为控制台和观众大屏两页，控制台负责切换轮次、开始滚动、喊停、确认中奖和导出结果；观众大屏只展示当前抽奖画面。

## 快速启动

项目不需要安装依赖，建议用本地静态服务器打开，确保控制台和大屏可以稳定同步。

```bash
cd /Users/jasonlau/Documents/projects/笃实抽奖
python3 -m http.server 5178
```

启动后打开：

- 控制台：http://localhost:5178/index.html
- 观众大屏：http://localhost:5178/screen.html

也可以在控制台点击 `大屏↗` 自动打开大屏页。

## 抽奖号码

号码池由日期自动生成，范围是：

- 开始日期：`2024-06-06`
- 结束日期：`2026-05-22`
- 总数量：`716` 个号码

页面展示时会以日期号码的形式滚动和翻牌，例如 `20260522`。

## 抽奖流程

当前流程共 6 个步骤：

| 步骤 | 奖项 | 默认人数 | 抽取方式 |
| --- | --- | ---: | --- |
| 第一轮 | 三等奖 | 40 | 象征性滚动后自动停止 |
| 第二轮 | 二等奖 | 20 | 象征性滚动后自动停止 |
| 第三轮 1/4 | 一等奖 · 第一抽 | 5 | 持续滚动，主控喊停 |
| 第三轮 2/4 | 一等奖 · 精选5项 | 5 | 持续滚动，主控喊停 |
| 第三轮 3/4 | 特等奖 | 1 | 持续滚动，主控喊停，可重复抽到满意为止 |
| 第三轮 4/4 | 超级特等奖 | 1 | 持续滚动，主控喊停，可重复抽到满意为止 |

第一轮和第二轮会自动滚动若干次后停在中奖结果上。第三轮开始后，点击 `开始滚动` 会一直滚动，再点击 `停止抽取` 才会定格结果。

## 现场使用方法

1. 启动本地服务器并打开控制台。
2. 点击 `大屏↗` 打开观众大屏，把大屏窗口投到现场屏幕。
3. 在控制台选择当前流程步骤，也可以用 `上一轮` / `下一轮` 切换。
4. 第一轮、第二轮点击 `开始抽取`，等待系统自动停下。
5. 第三轮点击 `开始滚动`，现场喊停时再点击 `停止抽取`。
6. 特等奖和超级特等奖会先进入待确认状态；如果结果不满意，点击 `继续滚动` 重新抽；满意后点击 `确认中奖`。
7. 抽奖结束后可点击 `⇩` 导出 CSV 结果。

## 控制说明

- `人数`：可调整当前步骤抽取人数。特等奖和超级特等奖固定为 1 人。
- `上一轮` / `下一轮`：切换抽奖步骤。
- `开始抽取`：第一轮、第二轮使用，自动滚动并停止。
- `开始滚动` / `停止抽取`：第三轮使用，支持现场喊停。
- `确认中奖`：用于确认特等奖和超级特等奖的当前结果。
- `‹` / `›`：当某轮结果超过一组时，切换大屏展示分组。
- `⛶`：让舞台区域进入或退出全屏。
- `⇩`：导出所有已确认中奖结果。
- `×`：清空本机保存的抽奖结果，正式使用前请谨慎操作。

## 大屏同步

控制台和大屏通过浏览器的 `BroadcastChannel` 和 `localStorage` 同步。建议控制台和大屏使用同一个浏览器、同一个本地地址打开，例如都使用 `http://localhost:5178/...`。

如果大屏没有同步，可以尝试：

1. 确认本地服务器仍在运行。
2. 刷新大屏页 `screen.html`。
3. 确认控制台和大屏打开的是同一个端口。

## 素材说明

主要素材在 `抽奖素材-1/` 目录中：

- `底图.png`：抽奖舞台底图。
- `位置1` 到 `位置5`：普通奖项五个位置的日期数字牌素材。

- `奖品贴图`：第三轮抽奖前展示的奖品背景图。

第三轮奖品图对应关系：

- `奖品贴图/third-1.png`：第三轮 1/4，一等奖 · 第一抽。
- `奖品贴图/third-3.png`：第三轮 3/4，特等奖。
- `奖品贴图/third-4.png`：第三轮 4/4，超级特等奖。

第三轮 2/4 当前不展示奖品图。

## 结果保存

抽奖状态会保存在当前浏览器的本地存储中。刷新页面不会丢失结果，但更换浏览器、清空浏览器数据或点击清空按钮都会影响本地结果。

建议正式活动前先清空一次旧结果，完成抽奖后立即导出 CSV 备份。
